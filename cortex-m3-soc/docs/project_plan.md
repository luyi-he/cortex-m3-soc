## 3. RTL 开发计划

### 第一阶段：基础模块 (W3)

| 模块 | 负责人 | 开始 | 结束 | 状态 |
|------|--------|------|------|------|
| clk_gen (时钟生成) | RTL 团队 | W3D1 | W3D2 | 🔄 |
| rst_gen (复位生成) | RTL 团队 | W3D1 | W3D2 | 🔄 |
| GPIO (4 端口) | RTL 团队 | W3D3 | W3D5 | ⏳ |

**后端约束确认**:
- 时钟定义：200MHz (HCLK), 100MHz (PCLK)
- 输入延迟：2ns (50MHz 外设)
- 输出延迟：2ns (GPIO)

### 第二阶段：总线模块 (W4)

| 模块 | 负责人 | 开始 | 结束 | 状态 |
|------|--------|------|------|------|
| AHB Matrix (1 主 3 从) | RTL 团队 | W4D1 | W4D3 | ⏳ |
| AHB2APB Bridge | RTL 团队 | W4D4 | W4D5 | ⏳ |

**关键时序路径**:
- AHB 读响应路径 (HREADY)
- 地址译码路径

### 第三阶段：存储模块 (W5)

| 模块 | 负责人 | 开始 | 结束 | 状态 |
|------|--------|------|------|------|
| Flash 控制器 | RTL 团队 | W5D1 | W5D3 | ⏳ |
| SRAM 控制器 (128KB) | RTL 团队 | W5D4 | W5D5 | ⏳ |

**后端关注**:
- SRAM 密度高，注意摆放位置
- Flash 接口时序严格

### 第四阶段：外设模块 (W6)

| 模块 | 负责人 | 开始 | 结束 | 状态 |
|------|--------|------|------|------|
| UART x2 | RTL 团队 | W6D1 | W6D2 | ⏳ |
| Timer x4 | RTL 团队 | W6D2 | W6D3 | ⏳ |
| WDT | RTL 团队 | W6D3 | W6D3 | ⏳ |
| NVIC 集成 | RTL 团队 | W6D4 | W6D5 | ⏳ |

### 第五阶段：顶层集成 (W6)

| 任务 | 开始 | 结束 | 状态 |
|------|------|------|------|
| 模块实例化 | W6D1 | W6D3 | ⏳ |
| 信号连接 | W6D3 | W6D4 | ⏳ |
| 综合检查 | W6D5 | W6D5 | ⏳ |

## 5. 后端协同流程

### 5.1 RTL→后端交接

```
RTL 完成 → Lint 检查 → CDC 检查 → 综合 → 时序评估 → 后端
    ↓                                        ↑
    └─────────── 迭代优化 ───────────────────┘
```

### 5.2 后端反馈检查点

| 检查点 | RTL 阶段 | 后端检查项 | 通过标准 |
|--------|----------|------------|----------|
| CP1 | 基础模块完成 | 时钟树规划 | 时钟 skew < 100ps |
| CP2 | 总线模块完成 | Floorplan 评估 | 拥堵率 < 70% |
| CP3 | 存储模块完成 | 电源网络规划 | IR drop < 10% |
| CP4 | 外设模块完成 | 时序预估 | WNS > -0.5ns |
| CP5 | 顶层集成完成 | 综合后时序 | WNS >= 0ns |

### 5.3 时序收敛策略

**如果时序不收敛**:

1. **RTL 级优化**
   - 流水线插入
   - 关键路径重定时
   - 状态机编码优化

2. **综合级优化**
   - 提高驱动强度
   - 调整阈值电压 (LVT/HVT)
   - 逻辑重组

3. **后端级优化**
   - 单元摆放优化
   - 时钟树优化
   - 缓冲器插入

### 5.4 拥堵预防

**高密度模块预警**:

| 模块 | 预估密度 | 缓解措施 |
|------|----------|----------|
| SRAM 控制器 | 高 | 靠近 SRAM 摆放 |
| NVIC | 中高 | 靠近 CPU 摆放 |
| GPIO | 中 | 沿芯片边缘摆放 |
| Flash 控制器 | 中 | 靠近 Flash 接口 |

**Floorplan 建议**:

```
+----------------------------------+
| GPIO PAD                         |
| +------------------------------+ |
| | AHB Matrix | SRAM            | |
| |------------|                 | |
| | CPU        | Flash Ctrl      | |
| | NVIC       |                 | |
| +------------+-----------------+ |
| | APB Bridge | Peripherals     | |
| +------------+-----------------+ |
| GPIO PAD                         |
+----------------------------------+
```

## 6. 风险评估

### 6.1 技术风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 时序不收敛 | 中 | 高 | 早期综合评估，预留 20% 时序余量 |
| 面积超标 | 低 | 中 | 模块化设计，面积监控 |
| 功耗超标 | 中 | 中 | 时钟门控，电源门控 |
| CDC 问题 | 中 | 高 | 严格 CDC 检查，同步器标准化 |
| 验证不充分 | 高 | 高 | 覆盖率驱动验证，回归测试 |

### 6.2 进度风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| RTL 延期 | 中 | 高 | 模块化并行开发，每周进度检查 |
| 验证延期 | 高 | 高 | 验证与 RTL 并行，早期介入 |
| 后端迭代 | 中 | 中 | 早期 Floorplan 评估 |
| 工具问题 | 低 | 中 | 备用工具链，EDA 支持 |

## 7. 质量检查点

### 7.1 RTL 质量门禁

- [ ] Lint 检查 0 错误 0 警告
- [ ] CDC 检查通过
- [ ] 代码覆盖率 >90% (模块级)
- [ ] 综合无 DRC 违例
- [ ] 时序 WNS > -0.5ns

### 7.2 验证质量门禁

- [ ] 所有 P0 测试用例通过
- [ ] 代码覆盖率 >95%
- [ ] 功能覆盖率 >95%
- [ ] 回归测试 100% 通过
- [ ] 无已知 bug

### 7.3 后端质量门禁

- [ ] 时序 WNS >= 0ns
- [ ] 时序 TNS < 100ps
- [ ] 密度 < 70%
- [ ] DRC 违例 < 10
- [ ] LVS 清洁
